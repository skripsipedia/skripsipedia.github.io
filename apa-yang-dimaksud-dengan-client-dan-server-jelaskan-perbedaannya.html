1. Apa yang di maksud dengan client dan server?jelaskan perbedaan keduanya!


Komputer server adalah komputer yang digunakan untuk menampung semua data yang berasal dari komputer client dalam sebuah jaringan. Tidak hanya itu, komputer server juga digunakan untuk mengelola sebuah jaringan komputer. Sirkulasi data, informasi, maupun perintah yang berasal dari komputer client akan selalu melewati komputer server ini.

Komputer client adalah komputer yang digunakan untuk meminta layanan tertentu dari komputer server. Layanan tersebut bisa jadi data, file, gambar, printer, maupun yang lainnya. Oleh karena itu, di dalamÂ komputer clientÂ dibutuhkan suatu aplikasi tertentu agar dapat mengakses layanan dari komputer server.

Perbedaannya :

Komputer server berfungsi untuk menyimpan seluruh data, melayani permintaan akses dari komputer client, serta mengelola lalu lintas data dalam sebuah jaringan. Sedangkan, komputer client meminta layanan dari komputer server untuk bisa mengakses sebuah data.

Komputer server dapat melakukan beberapa layanan yang dibutuhkan komputer client secara bersamaan. Sedangkan, komputer client tidak mempunyai layanan yang dapat diakses oleh komputer client lain.

Client dan Server merupakan sebuah teknologi jaringan dimana Server bertugas melayani permintaan data dari Client, Server merupakan pusat pengolahan data sedangkan Client menyediakan interface yang berhubungan langsung dengan User yang selanjutnya meneruskan permintaan data kepada Server

Pembahasan

Client-Server merupakan suatu pembagian kerja antara server maupun client yang mengakses server dalam suatu jaringan. Dengan demikian, arsitektur client-server merupakan desain sebuah aplikasi yang terdiri dari client dan server yang saling berkomunikasi ketika mengakses server dalam suatu jaringan. Ada beberapa ciri-ciri client dan server, antara lain:

Servis atau layanan, yang merupakan hubungan antara proses yang berjalan pada mesin yang berbeda. Sharing resources atau sumber daya, untuk melayani beberapa client pada waktu yang sama dan meregulasi akses bersama dalam share sumber daya dalam menjamin konsistensinya. Asymmetrical protocol atau protokol yang tidak simetris, many to one relationship antara client dan server. Dimana client selalu menginisiasikan dialog melalui layanan permintaan bahkan server menunggu secara pasif request dari client. Transparansi lokasi, merupakan proses yang dilakukan server yang terletak pada mesin yang sama bahkan pada mesin yang berbeda melalui jaringan atau lokasi server harus muda di akses dari client.
