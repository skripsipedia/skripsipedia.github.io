3. apa perbedaan antara pemrograman prosedural dan pemrograman berbasis objek


Pemrograman Prosedural

Pemrograman Prosedural adalah dilakukan dengan memberikan serangkaian perintah yang berurutan. Paradigma ini didasari oleh konsep mesin Von Newman (stored program concept) sekelompok tempat penyimpanan (memori), yang dibedakan menjadi memori instruksi dan memori data, masing-masing memori tersebut dapat diberi nama dan nilai, selanjutnya instruksi akan dieksekusi satu persatu secara sekuensial oleh sebuah proses tunggal.

Program dalam paradigma ini berdasarkan pada struktur informasi di dalam memori dan manipulasi dari informasi yang disimpan tersebut. Kata kunci yang sering digunakan dalam paradigma ini adalah:

Algoritma + Struktur Data = Program

Pemrograman berorientasi objek (PBO)

Pemrograman berorientasi objek merupakan paradigma pemrograman yang berorientasikan kepada objek. Semua data dan fungsi di dalam paradigma ini dibungkus dalam kelas-kelas atau objek-objek. Bandingkan dengan logika pemrograman terstruktur. Setiap objek dapat menerima pesan, memproses data, dan mengirim pesan ke objek lainnya.

Pengertian inheritance pada pemrograman berbasis objek

Inheritance atau Pewarisan/Penurunan adalah konsep pemrograman dimana sebuah class dapat ‘menurunkan’ property dan method yang dimilikinya kepada class lain. 

• Jawaban panjang: 

Inheritance
Inheritance atau Pewarisan/Penurunan adalah konsep pemrograman dimana sebuah class dapat ‘menurunkan’ property dan method yang dimilikinya kepada class lain. Konsep inheritance digunakan untuk memanfaatkan fitur ‘code reuse’ untuk menghindari duplikasi kode program. Konsep inheritance membuat sebuah struktur atau ‘hierarchy’ class dalam kode program. Class yang akan ‘diturunkan’ bisa disebut sebagai class induk (parent class), super class, atau base class. Sedangkan class yang ‘menerima penurunan’ bisa disebut sebagai class anak (child class), sub class, derived class atauheir class.

27. jelaskan bagaimana pengembangan pemrograman berbasis platform berbeda dibandingkan pemrograman secara umum ? mohon dibantu para warga milenial


Dilihat dari sisi metode pengembangannya, pemrograman berbasis multiplatform terlebih mobile, dibagi menjadi Tiga yaitu Native, Hybrid, dan PWA (Progressive Web Application). Tetapi yang sudah "matang" meluncur di dunia IT adalah Native dan Hybrid, maka yang akan dibahas hanya Native dan Hybrid saja (karena secara garis besar PWA hampir sama dengan Hybrid)

Native :

Pemrograman yang terfokus pada satu platform saja, misalnya kalau kamu mau membuat aplikasi berbasis Windows, maka bahasa pemrograman yang harus kamu kuasai adalah Basic untuk mengembangkan aplikasi dengan Visual Basic, tetapi kamu tidak bisa membuat aplikasi berbasis Linux atau OSX dengan Visual Basic, kamu membutuhkan bahasa pemrograman yang berbeda lagi.

Dalam Mobile, Jika kamu ingin mengembangkan aplikasi basis Android, maka kamu harus mempelajari bahasa pemrograman Java (sekarang berkembang ada Kotlin, Golang). Tetapi tidak bisa untuk mengembangkan aplikasi iOS, untuk iOS kamu harus menggunakan bahasa yang berbeda lagi, seperti Swift.

Kelebihan Native, adalah fleksibilitas tanpa batas, oleh karena bahasa yang digunakan memang sudah diatur agar sesuai dengan platform yang akan digunakan, serta dukungan library yang luas, karena lebih mudah fokus ke satu platform saja.

Hybrid :

Pemrograman yang bisa berjalan di lebih dari satu platform (multiplatform), seperti Pascal, kamu bisa membuat aplikasi Windows menggunakan Lazarus dengan bahasa Pascal, tetapi kamu juga bisa membuat aplikasi Linux menggunakan aplikasi dan bahasa pemrograman yang sama.

Dalam mobile, Jika kamu ingin mengembangkan aplikasi basis Android maupun iOS, atau bahkan Blackberry dan Windows Phone sekalipun, kamu hanya perlu menggunakan HTML5, aplikasi yang dikembangkan menggunakan HTML5 akan bekerja di semua platform tersebut selama masih didukung.

Kelebihan Hybrid, jelas. kamu tidak perlu belajar bahasa pemrograman banyak untuk banyak platform. Dibanding Native, Hybrid hanya tidak bisa se fleksibel Native, karena ruang kendali yang terbatas oleh "container" yang menterjemahkan setiap perilaku yang dilakukan oleh aplikasi Hybrid ini.

37. kelebihan dan kekurangan pemrograman berbasis objek


Dalam pemrograman berbasis objek, setiap objek dapat menerima pesan, mengirim pesan dan memproses data ke objek lain. Pendekatan berorientasi objek merupakan paradigma pemrograman berbasis objek yang memasukkan data berupa file, biasa disebut dengan kode prosedur atau biasa disebut metode. Masalah juga terlihat pada sistem perangkat lunak.

Pembahasan

Ada beberapa kelebihan di sini.

Program dibagi menjadi objek sehingga objek yang sama atau berbeda dapat dengan mudah dimodifikasi dan objek baru dibuat.Jika Anda sedang membangun perpustakaan atau kerangka kerja, OOP sangat bagus karena strukturnya yang rapi sehingga memudahkan untuk debug, mengubah kode, dll. Dataset private hanya dapat diakses oleh fungsi lokal, bukan eksternal, jadi Anda tidak perlu mengatur variabel dengan banyak nama dalam fungsi yang berbeda karena takut terpengaruh oleh efek lain.gaya C++. Banyak programmer yang memulai di C++ beralih ke pemrograman berbasis objek (Java) karena sintaks untuk pemrograman di Java sama persis dengan pemrograman di C++.Fungsi manajemen memori sudah ada di dalamnya, sehingga programmer tidak perlu khawatir tentang pengelolaan penggunaan memori.Semua aspek Java berorientasi objek. Dalam hal ini, karena semuanya didasarkan pada objek atau kelas, menulis program sangat mudah, sehingga lebih mudah untuk merancang, menemukan bug, dan mengembangkan program yang telah direncanakan sebelumnya.Multi-platform (dapat berjalan pada sistem operasi yang berbeda secara bersamaan).

berikut beberapa kekurangannya

Ini menghabiskan banyak memori karena ada begitu banyak program dan mereka menghabiskan banyak memori.pembongkaran sederhana.Benar jika semua program harus diedit dengan benar
